# Identity & Access Management

## Overview

Identity is the foundation of this lab. Rather than treating authentication as a standalone service, identity is designed to integrate with infrastructure, networking, and security monitoring to better reflect a modern enterprise environment.

The goal is to gain practical experience designing, managing, and securing identity services while applying least-privilege principles and modern authentication practices.

---

## Objectives

* Deploy an enterprise-inspired Active Directory environment.
* Integrate Microsoft Entra ID with on-premises identity.
* Develop practical experience with Group Policy.
* Implement role-based administration.
* Explore identity-related security monitoring and auditing.

---

---

## Security Principles

The identity environment is built around several core principles.

### Least Privilege

Administrative access should only be granted when required.

---

### Separation of Duties

Administrative responsibilities should be separated wherever practical.

---

### Secure Authentication

Strong authentication mechanisms should protect privileged accounts.

---

### Centralized Identity

Authentication should be centrally managed whenever possible to improve consistency and auditing.
